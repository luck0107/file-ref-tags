## 实现差异

### UI 交互方式

| 功能 | VSCode 版本 | JetBrains 版本 |
|------|------------|----------------|
| 编辑标题 | 鼠标悬浮显示编辑按钮 | 右键菜单选择编辑 |
| 删除引用项 | 鼠标悬浮显示删除按钮 | 右键菜单选择删除 |
| 添加分组 | 工具栏按钮 | 工具栏按钮 |
| 查看存储位置 | 工具栏按钮 | 工具栏按钮 |

### 技术实现

| 方面 | VSCode 版本 | JetBrains 版本 |
|------|------------|----------------|
| 开发语言 | TypeScript | Kotlin |
| UI 框架 | Webview (HTML/CSS/JS) | Swing (JBList/JPanel) |
| 数据存储 | JSON 文件 | JSON 文件 |
| 存储位置 | VSCode 全局存储目录 | IntelliJ 配置目录 |
| 协议支持 | vscode:// | 不支持 |

## 功能完整性

✅ **所有核心功能已完全实现**

JetBrains 版本已经实现了 VSCode 版本的核心功能，包括：
- 所有添加引用项的功能
- 完整的分组管理功能
- 所有管理功能（编辑、删除、排序、跳转）
- 数据持久化

**注意**：JetBrains 版本不支持深度链接功能（包括复制和接收外部链接），VSCode 版本支持。

## 平台特定优化

### JetBrains 版本的优势

1. **更好的 IDE 集成**：使用原生 Swing 组件，与 IDE 界面风格一致
2. **更快的响应速度**：原生组件，无需 Webview 渲染
3. **更好的性能**：直接使用 IntelliJ Platform API，性能更优

### VSCode 版本的优势

1. **更灵活的 UI**：使用 Webview，可以自定义样式
2. **更丰富的交互**：鼠标悬浮显示操作按钮

## 总结

JetBrains 版本已经实现了 VSCode 版本的核心功能。主要差异是：
- **JetBrains 版本不支持深度链接功能**（包括复制和接收外部链接，VSCode 版本支持）

两个版本在核心功能上对等，用户可以根据自己的 IDE 选择使用相应的版本。如果需要深度链接功能，请使用 VSCode 版本。
